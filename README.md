# 생기부 문장 생성 도우미

선생님이 학생별 활동 기록을 바탕으로 생활기록부 문장 초안을 빠르게 만들 수 있도록 돕는 웹앱입니다.
과세특, 동아리 세특, 행동특성 및 종합의견, 가정통신문 문장을 한 화면에서 작성하고 확인할 수 있습니다.

AI가 만든 문장은 그대로 제출하는 최종본이 아니라, 선생님이 검토하고 다듬기 위한 초안입니다.

## 이런 선생님께 도움이 됩니다

- 학생 수가 많아 세특 초안을 빠르게 정리해야 하는 선생님
- 같은 활동을 했지만 학생별 성취도와 참여 정도를 다르게 반영하고 싶은 선생님
- 동아리 활동, 행동특성, 가정통신문 문장을 자연스럽게 시작하고 싶은 선생님
- 엑셀 명단을 활용해 여러 학생의 결과를 한 번에 관리하고 싶은 선생님
- 로컬 LLM, OpenAI, NVIDIA 등 여러 AI 모델을 상황에 맞게 선택하고 싶은 선생님

## 처음 사용하는 순서

1. 학생 수를 선택합니다.
2. 과목명, 동아리명, 관찰 내용 등 필요한 기본 정보를 입력합니다.
3. 공통 활동과 학생별 개별 활동을 입력합니다.
4. 필요하면 활동별 성취도나 추가 지침을 선택합니다.
5. `AI 생성`을 누릅니다.
6. 결과 문장을 읽고 학생 사실과 학교 기준에 맞게 수정합니다.
7. 필요한 경우 엑셀로 내려받아 보관합니다.

## 화면별 기능 안내

### 1. 과세특(자유학기 세특)

과목 시간에 진행한 활동을 바탕으로 과목별 세부능력 및 특기사항 문장을 만듭니다.

할 수 있는 일:

- 과목명과 활동 내용을 입력합니다.
- 여러 활동을 순서대로 입력합니다.
- 학생별 개별 활동 내용을 따로 입력합니다.
- 활동별 성취도를 A부터 E까지 선택합니다.
- 1500byte, 1000byte, 600byte, 직접 입력 중 원하는 분량을 고릅니다.
- 생성 결과를 학생별로 확인하고 복사합니다.
- 결과를 엑셀 파일로 내려받습니다.

활동별 성취도 기준:

- A: 매우 잘함. 주도성, 심화 탐구, 구체적 성과가 드러나도록 작성합니다.
- B: 잘함. 안정적인 수행, 핵심 이해, 성실한 참여가 드러나도록 작성합니다.
- C: 보완 필요. 일부 기준점에 아직 충분히 도달하지 못한 부분을 부드럽게 드러냅니다.
- D: 많은 보완 필요. 반복적인 안내와 많은 보완이 필요한 지점을 C보다 더 분명하게 씁니다.
- E: 매우 많은 보완 필요. 기본 활동 수행에 어려움이 크고 지속적인 지도가 필요함을 조심스럽게 씁니다.

주의할 점:

- C, D, E는 점점 더 보완이 필요한 방향으로 작성되지만, 학생을 비난하거나 낙인찍는 표현은 피하도록 안내되어 있습니다.
- 과목명은 AI가 참고만 하며, 결과 문장에는 직접 들어가지 않도록 제한되어 있습니다.
- 학생이 실제로 하지 않은 활동은 입력하지 않는 것이 좋습니다.

### 2. 동아리 세특

동아리 활동 과정, 협력, 역할 수행, 탐구 과정을 바탕으로 동아리 특기사항 문장을 만듭니다.

할 수 있는 일:

- 동아리명과 공통 활동을 입력합니다.
- 학생별로 다르게 참여한 활동을 따로 입력합니다.
- 활동이 많을 때 추가 지침으로 일부 활동만 반영하도록 안내할 수 있습니다.
- 고등학교를 선택하면 더 심화된 탐구 과정과 성장 흐름을 반영합니다.

작성 방향:

- 활동명을 단순히 반복하지 않고 자연스럽게 문장을 시작하도록 안내합니다.
- 문제의식, 자료 조사, 실험, 토의, 발표, 협업 등 활동 성격에 따라 시작 방식이 달라지도록 구성되어 있습니다.
- 학생별 개별 활동이 있더라도 공통 활동 흐름 안에 자연스럽게 녹여 씁니다.

### 3. 행동특성 및 종합의견

학생의 생활 태도, 관계, 책임감, 배려, 성장 가능성을 바탕으로 행동특성 및 종합의견 문장을 만듭니다.

할 수 있는 일:

- 학생에게서 관찰한 행동을 입력합니다.
- 학교급에 맞는 문장 수준으로 생성합니다.
- 고등학교를 선택하면 구체적 장면과 지속적으로 관찰된 태도를 더 강조합니다.
- 웹 검색 보강을 사용할 경우에도 입력한 관찰 내용을 우선하도록 안내합니다.

작성 방향:

- 실제 관찰된 행동을 중심으로 씁니다.
- 지나치게 추상적인 칭찬보다 구체적인 장면을 반영하도록 돕습니다.
- 부정적인 내용은 교사용 기록 문장으로 조심스럽게 바꾸는 것을 목표로 합니다.

### 4. 가정통신문

학기말이나 방학 전 학부모님께 보낼 안내 문장을 만듭니다.

할 수 있는 일:

- 학업, 건강, 친구관계, 가족관계 같은 안내 키워드를 입력합니다.
- 여름방학, 겨울방학, 새 학기 등 상황에 맞는 문장을 생성합니다.
- 생활기록부 문체가 아니라 학부모님께 전달하기 좋은 경어체로 작성합니다.

작성 방향:

- 학생 평가와 방학 생활 조언이 자연스럽게 이어지도록 구성합니다.
- 키워드를 단순 나열하지 않고 문장 흐름 안에 넣습니다.
- 입력하지 않은 구체 활동을 지어내지 않도록 안내되어 있습니다.

## 공통 기능

### 학생 명단 관리

- 학생 수를 직접 선택할 수 있습니다.
- 엑셀 명렬표를 업로드해 학생 이름을 불러올 수 있습니다.
- 학생별 결과를 각각 다시 생성하거나 지울 수 있습니다.

### 추가 지침

`추가 지침 사항`에는 AI가 꼭 지켜야 할 조건을 적습니다.

예시:

```text
활동 내용 중 4개를 랜덤으로 선택해 작성해 주세요.
학생별 개별 활동 내용을 반드시 반영해 주세요.
축구는 단체 경기 결과보다 개인별 역할과 태도를 중심으로 작성해 주세요.
```

### 웹 검색 보강

학생별 개별 활동 내용을 검색해 작품, 논문, 연구, 쟁점의 배경을 보강할 수 있습니다.

주의할 점:

- 검색 자료는 배경 이해용입니다.
- 학생이 실제로 한 활동보다 검색 결과를 우선하면 안 됩니다.
- 검색 보강을 사용하려면 `.env`에 Tavily API 키가 필요합니다.

### 엑셀 내려받기

생성된 결과는 엑셀 파일로 저장할 수 있습니다.

과세특에서는 다음 내용이 함께 저장됩니다.

- 번호
- 성명
- 활동별 성취도
- 세부능력 및 특기사항

## AI 모델 안내

기본 로컬 모델은 `Gemma 4 12B`입니다.
앱 화면에서 사용할 AI 모델을 선택할 수 있습니다.

사용 가능한 주요 경로:

- 로컬 LLM: 기본 생성 경로
- OpenAI: API key를 입력하면 OpenAI 모델로 생성
- NVIDIA NIM: 환경 변수 설정 시 사용 가능

초보자 선생님은 특별한 설정을 바꾸지 않고 기본값으로 먼저 사용해도 됩니다.

## 실행 방법

개발용으로 직접 실행하려면 Node.js가 필요합니다.

```bash
npm install
npm run dev
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:3000
```

## 환경 설정

검색 보강을 사용하려면 `.env`에 Tavily 설정이 필요합니다.

```env
TAVILY_API_URL=https://api.tavily.com/v1
TAVILY_API_KEY=your_tavily_api_key
```

원격 LM Studio 서버를 사용할 때는 아래 값을 확인합니다.

```env
LMSTUDIO_API_URL=https://lm.alluser.site
LMSTUDIO_API_KEY=your_api_key
```

API 키가 들어 있는 `.env` 파일은 GitHub에 올리면 안 됩니다.

## 결과를 확인할 때 꼭 볼 것

- 학생 이름이 결과 문장 안에 들어가 있지 않은지 확인합니다.
- 과목명이나 프로그램명이 결과 문장에 직접 들어가 있지 않은지 확인합니다.
- 실제로 하지 않은 활동이 추가되지 않았는지 확인합니다.
- 문장이 너무 칭찬 위주이거나 너무 부정적으로 치우치지 않았는지 확인합니다.
- C, D, E 성취도 문장은 보완 필요가 드러나되 학생을 낙인찍지 않는지 확인합니다.
- 학교생활기록부 기재 기준과 학교 내부 표현 기준에 맞는지 최종 검토합니다.

## 개발자용 확인 명령어

```bash
npm test
npm run build
```

- `npm test`: 생성 규칙, 모델 호출, 검색 보강, 화면 연결 테스트를 실행합니다.
- `npm run build`: 배포 가능한 Next.js 앱으로 빌드되는지 확인합니다.

## 폴더 구조

```text
app/              화면과 API 라우트
components/       공통 UI 컴포넌트
utils/            생성, 검증, 검색, 엑셀 처리 로직

```

## 마지막 안내

이 앱은 선생님의 기록 업무를 줄이기 위한 도구입니다.
최종 판단은 항상 선생님이 해야 하며, 생성 결과는 학생의 실제 활동과 학교 기준에 맞게 직접 확인한 뒤 사용해 주세요.
